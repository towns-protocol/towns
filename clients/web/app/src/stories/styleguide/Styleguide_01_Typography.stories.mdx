# Typography

## Body Text

<Box>
  {["sm", "md", "lg"].reverse().map((c) => (
    <Row key={c} label={`${c}${c === "md" ? ` (default)` : ``}`}>
      <Paragraph size={c}>Paragraph {c}</Paragraph>
    </Row>
  ))}
</Box>

```tsx
<!-- customized body text -->
<Paragraph size="lg">Text content</Paragraph>

<!-- used in uncontrolled html -->
<Box>
  <p>Text content</p>
</Box>
```

## Button Text

<Box>
  {["sm", "md", "lg"].reverse().map((c) => (
    <Row key={c} label={`${c}${c === "md" ? ` (default)` : ``}`}>
      <ButtonText size={c}>Button Text {c}</ButtonText>
    </Row>
  ))}
</Box>

```tsx
<ButtonText size="xl">Menu Item</ButtonText>
```

## Headings

<Box>
  {[1, 2].map((c) => (
    <Row key={c} label={`${c}${c === 1 ? ` (default)` : ``}`}>
      <Heading level={c}>Heading level {c}</Heading>
    </Row>
  ))}
</Box>

```tsx
<!-- customized heading -->
<Heading level={3} color="critical">Text content</Heading>

<!-- used in uncontrolled html -->
<Box>
  <h1>Title Text</h1>
</Box>
```

## Colors

<Box>
  {Object.keys(vars.color.text).map((c) => (
    <Row
      key={c}
      label={c}
      backgroundVariant={
        c.match(/inverted/i)
          ? "inverted"
          : c.match(/onTone/)
          ? "accent"
          : undefined
      }
    >
      <Paragraph color={c}>{c}</Paragraph>
    </Row>
  ))}
</Box>

```tsx
<Paragraph color="gray2">Text content</Paragraph>

<!-- inherited -->
<Box color="gray2">
  <p>Title Text</p>
</Box>
```

## Text Primitive

The `Text` primitive is less opinionated and restrictive than `Paragraph` and `Heading`.
It's a perfect fit to build custom text components.

**Note:** Make sure to import the component. Because `<text>` is a native HTML tag, TSC wont warn if the module hasn't been imported and React will still render the content (unstyled)

<Box border padding>
  <Text as="span" color="gray2" size="lg" textTransform="uppercase">
    Custom Text
  </Text>
</Box>

```tsx
<Text as="span" color="gray2" size="lg" textTransform="uppercase">
  Custom Text
</Text>
```

## Cap Size Trimming

Top and bottom are cut thanks to negative margin applied on pseudo objects. This
enables accurate and predictable spacing between blocks and typography.
Inspired by - https://github.com/seek-oss/capsize
